**Mapping Mission and Business Functions to Locations, Assets, Technologies etc:**



Mission or Business Functions are important to define before a security or safety event occurs. Below are some steps organizations should take to figure out their mission dependencies left of boom. 



Step 1) at the overall Org level what is it that your organization does and how? What are the top 10 functions of the entire organization?

Step 2) move down to the various departments, divisions, business units, brand or regional segments, etc. what are the top 10 functions for each of them.

Step 3) move down to a site specific level. What are the top 10 functions. What do they exist to make or do etc?

Step 4) move down to the process, zone and system levels. What are the top 10 functions for each?

Step 5) now build a catalog of each of the top 10 functions at each level for each entity.

Step 6) what processes enable or achieve each function at each level?

Step 7) what roles, disciplines, providers, suppliers, services and technologies support, execute, enable each process and each function at each level?

Step 8) build a catalog list of various types of impacts mapped to each individual function at each level.

Step 9) assign each of those types of impacts a severity level.

Step 10) build a catalog of internal as well as upstream, downstream and potential external consequences that could be caused by each impact in the catalog that is mapped to each function at each level.

Step 11) define severity levels for each consequence in the catalog that is linked to each respective source impact that would trigger or enable that consequence.

Step 12) for each consequence that is linked to each impact that is linked to each respective function at each level, define a mitigation to prevent, minimize and or contain the consequence linked to each impact linked to each respective function at each level.

step 13) build a heat map based on those severity levels so you can visualize each type of consequence mapped to each respective impact mapped to each respective function at each level.

Step 14) if you want you can also use a tool like mindmapper to start at step 1 and build a visual mind map of steps 1-12. You can also build the entire catalog of functions at each level and their respectively mapped impacts, consequences, severity levels and mitigations into a spreadsheet or a tool like smartsheets.



Use that catalog data, the heat map and the mindmap to build a visual story board so you can see how those functions, impacts, consequences etc change over time through mergers, acquisitions, divestments etc etc.



You can also add funding numbers to the functions, impacts, consequences and mitigations.



You can use this visualization and data to look for patterns like common functions, impacts, consequences and mitigations. You can look for examples of where severity levels change based on which level and type of function is involved etc. You can do the same for mitigations when you discover which mitigations can prevent, minimize or contain more than one consequence and impact at which severity levels and function levels etc.
